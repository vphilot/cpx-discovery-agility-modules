Intro

This is an outline of our most urgent needs in terms of content updates. Being able to modularize and componentize these sections would empower content editors to make changes on the fly and enable us to have more time to work on other higher priority projects.

Please notice that we're only focusing on the most frequently updated items on this document. There are still several other opportunities for modularization in the long run.

Suggested common specs for all new modules:

  • Have customizable classes for every single item, including parents and children
  • Avoid bootstrap classes so we can have flexibility and avoid implementing overrides
  • Avoid unnecessary nesting as much as possible
  • Need to meet basic accessibility requirements out of the box
  • Need to be responsive and mobile-first
  • Must prioritize pulling information from our API and databases and avoid manual entry to keep consistency throughout our ecosystem

Theatre (VIP) Menu

This menu page was redesigned in 2019 but the intensive workload required to manually update, add, and remove menu items implies the need to modularize it. Please refer to the VIP Page to see the current implementation.

  • Structure: Menu categories (e.g. Entrées) > Menu subcategories (e.g. Poutines) > Menu items
  • A single menu item contains:
    • Item title
    • (optional) item quantity / volume
    • Item description
    • Pricing
    • (optional) item modifiers (e.g. Add Bacon)
    • (optional) item modifiers pricing
    • (optional) Vegetarian, Vegan, and/or Gluten Free tags
  • Content editors should be able to add / remove seasonal and fixed items as the menu changes
